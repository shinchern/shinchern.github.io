Flow* Toolbox -
A Platform for the Formal Analysis of Cyber-Physical Systems



Introduction

This site is created for the toolbox version of Flow* which is a formal verification tool for Cyber-Physical Systems based on reachability analysis. The first version of Flow* was released in 2013 as a command line application that accepts a text file defining a reachability problem on a CPS described as a hybrid automaton. The tool has been updated several times to the latest version 2.1.0 which is available at flowstar.org.

The main purpose to release a toolbox version of Flow* is to provide a more flexible way for users to build and model their safety verification problems. The toolbox version allows to directly call the C++ API and build new algorithms based on the basic functions in the tool.

Q:   How to use the Flow* toolbox?
A:    The Flow* toolbox provides a set of C++ data structures and functions for modeling the continuous and discrete dynamics in a CPS. Unlike the Flow* tool version, the toolbox focuses on providing a platform that allows users to build CPS simulators generating reachable set overapproximations using C++ programming language. Q:   What are the guarantees on the obtained results?
A:   The reachable set overapproximations are guaranteed to contain all system reachable states. Not only the approximation errors but also all numerical errors due to roundoff, truncation and cutoff in the reachability computation are included in the overapproximations.

Q:   Is there any update from Flow* 2.1.0 in the toolbox?
A:   All basic data structures are updated in the toolbox, and the performance is improved by at least 80% from Flow* 2.1.0.

You may write a simulator for your CPS using the C++ API provided by the toolbox. Instead of giving a comprehensive user's manual, we introduce the template programs for the following classes.

  • Continuous system defined by linear ODE.
  • Continuous system defined by nonlinear ODE.
  • Dealing with a time-triggered jump.
  • Dealing with an event-triggered jump.
